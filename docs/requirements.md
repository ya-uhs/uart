# Avalonia シリアル通信GUIアプリ 要件定義書

## コンセプト

**組み込みエンジニア向けの、マルチプラットフォーム対応シリアル通信ツール**

TeraTerm・PuTTY・CoolTermといった既存ツールはWindows専用またはUIが古く、
「マクロ送信」「HEX表示」「ログ解析」を一つのアプリで完結できるものがない。
これを Avalonia UI で解決する。

---

## ターゲットユーザー

- 組み込みエンジニア（自分自身を含む）
- マイコン・FPGAのデバッグ・検証を日常的に行う開発者
- Windows / macOS / Linux を混在して使う環境にいる人

---

## 機能要件

### v1.0（最初のリリース）

#### 接続管理
- COMポート / シリアルデバイスの自動検出・一覧表示
- ボーレート・データビット・パリティ・ストップビット・フロー制御の設定
- 接続 / 切断のワンクリック操作
- 接続状態の視覚的フィードバック（インジケーター）

#### 送受信ターミナル
- **HEX / ASCII の切替表示**（同時表示も可能にする）
- タイムスタンプ付きログ表示
- 改行コード設定（CR / LF / CRLF）
- 受信データのスクロール制御（オートスクロール ON/OFF）
- 表示バッファのクリア

#### マクロ・コマンド送信
- よく使うコマンドをボタンとして登録（マクロボタン）
- コマンド名・送信文字列・改行コードをマクロごとに設定
- マクロのインポート / エクスポート（JSON形式）
- 送信履歴の保存・呼び出し（↑↓キー）
- シーケンス送信（コマンドをリスト化してインターバル付きで順次送信）

#### セッション管理
- 接続設定・マクロ設定をセッションとして保存・復元
- 複数セッションの切替

---

### v2.0以降（将来対応）

#### ログ・エクスポート
- 受信ログをCSV / TXTで保存
- タイムスタンプ付きエクスポート

#### リアルタイムグラフ（ScottPlot）
- 受信データから数値を自動検出してプロット
- 表示範囲のデータのみ描画（LODによる間引き）
- バッファリング + 16ms間隔更新（60fps相当）

#### 送受信トリガー
- 特定文字列の受信をトリガーに自動コマンド送信
- 条件・アクションのルール設定UI

#### プロトコル解析
- パケット区切り設定（STX/ETX・改行・バイト数）
- チェックサム自動計算・検証

## 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | 115200bps受信時にUIが詰まらないこと |
| スレッド設計 | 受信はバックグラウンドスレッド、UI更新はDispatcher経由 |
| UIテーマ | ダークモード対応 |
| 起動速度 | 3秒以内 |

---

## 技術スタック

| 項目 | 採用技術 |
|------|----------|
| UIフレームワーク | Avalonia UI |
| アーキテクチャ | MVVM（CommunityToolkit.Mvvm） |
| シリアル通信 | System.IO.Ports |
| グラフ（v2以降） | ScottPlot |
| スレッド間通信 | ConcurrentQueue + DispatcherTimer |
| 設定保存 | JSON（System.Text.Json） |
| ターゲット | .NET 8 |

---

## スレッド設計方針

```
シリアル受信スレッド（DataReceived）
    ↓ ConcurrentQueue に積む
DispatcherTimer（16ms = 60fps）
    ↓ UIスレッドでまとめてターミナル・グラフを更新
Dispatcher.UIThread.InvokeAsync()
```

受信のたびにUIを叩かず、タイマーでバッチ処理することでUIの詰まりを防ぐ。
